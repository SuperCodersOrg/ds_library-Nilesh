### String Hashing

Implemented string hashing using:

```cpp
hashValue = hashValue * 31 + static_cast<unsigned char>(ch);
```

Learned:

* Polynomial rolling style hashing.
* How character positions influence the final hash.
* Why multiplication is necessary instead of simple addition.
* Why 31 is commonly used:

  * Prime number.
  * Good practical distribution.
  * Low collision rate.
  * Historically used in Java's String hash implementation.
  * Can be optimized as:

    ```cpp
    (hash << 5) - hash
    ```

Also learned that other multipliers such as 33, 37, 53, and 131 can be used, but 31 is a well-tested practical choice.

## Key Takeaways

* HashMaps are fundamentally arrays of buckets combined with hash functions.
