# 2026-06-23

## Overview

Today I focused on implementing a generic `DynamicArray<T>` from scratch using manual memory management. Most of the work involved understanding object lifetime, exception safety, and handling both primitive and non-primitive types correctly.

## Work Completed

* Implemented DynamicArray constructors and destructor.
* Added dynamic resizing support.
* Implemented `push_back()` and `pop_back()`.
* Added bounds-checked `operator[]` overloads (const and non-const).
* Added helper functions for object destruction and memory cleanup.
* Added exception handling for allocation failures and invalid accesses.
* Started designing copy constructor support (Rule of Three).

## Problems Faced

### 1. Managing Non-Primitive Types with malloc()

**Issue:**
`malloc()` allocates raw memory but does not construct objects.

**Resolution:**
Learned and implemented placement new:

```cpp
new (&arr[i]) T(value);
```

to explicitly construct objects inside raw memory.

---

### 2. Proper Cleanup During Exceptions

**Issue:**
If object construction failed midway, already constructed objects would leak resources.

**Resolution:**
Implemented cleanup helpers and exception-safe construction logic using:

```cpp
catch(...)
```

followed by manual destruction and memory release.

---

### 3. Destructor Handling for Generic Types

**Issue:**
Primitive types do not need destructor calls, but non-primitive types do.

**Resolution:**
Used:

```cpp
std::is_trivially_destructible_v<T>
```

with `if constexpr` to selectively call destructors only when required.

---

### 4. Bounds Safety

**Issue:**
Array access could lead to undefined behavior.

**Resolution:**
Added bounds checking using:

```cpp
std::out_of_range
```

inside `operator[]`.

---

### 5. pop_back() on Empty Array

**Issue:**
Removing an element from an empty array caused invalid behavior.

**Resolution:**
Added underflow protection using:

```cpp
std::underflow_error
```

---

### 6. Unnecessary Copies in push_back()

**Issue:**
Passing values by copy introduced extra object copies.

**Resolution:**
Changed:

```cpp
push_back(T value);
```

to:

```cpp
push_back(const T& value);
```

to improve efficiency.
